# Contributing to MicroClaw

Thanks for your interest! A few guidelines:

## How to contribute

1. **Fork** the repo — don't push directly to `master`
2. Create a feature branch: `git checkout -b feature/your-idea`
3. Make your changes, test them
4. Open a **Pull Request** against `master` with a clear description

## What we're looking for

- Bug fixes with reproduction steps
- New skills or routines (see `skills/` and `routines/` for the format)
- Improvements to the CLI or API
- Docker / deployment improvements
- Documentation

## What to avoid

- Direct pushes to `master`
- Breaking changes to the `SKILL.md` / `ROUTINE.md` format without discussion
- Adding heavy dependencies without a good reason (keep it micro)

## Questions

Open an issue or start a discussion on GitHub.
